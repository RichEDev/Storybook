Framework Parameters
Framework Parameter - What are they?
Framework contains system wide parameters that permit a change in appearance or functionality.

If any of the settings are modified, the action will affect the application as a whole and therefore every user of the system.

The parameters are location specific, so it is possible to have different settings in different database locations (compartments).

The relevant location should be selected from the filter drop-down list at the top of the screen.

Caution should be taken before considering whether to change any of the parameters. If in doubt, contact Software Europe who will be happy to answer any questions you may have.

For details of parameters available for modification, see Parameter Definitions section below.
 Parameter Definitions
Parameter Description
CONCAT_MANDATORY Specify whether the selection of a contract category is a mandatory field in contract details.
0 = DISABLED
1 = ENABLED (default)
CONCAT_TITLE Permits the renaming of the Contract Category field should the use need to be altered from its default.
CONDESC_TITLE Permits the renaming of the Contract Description field should the use need to be altered from its default.

Note: This parameter is used when client resolution is at least 1024x768.
Must update CONDESC_SHORT_TITLE for when resolution is below this.
CONDESC_SHORT_TITLE See above.
CONTRACT_NUM_GEN Activate the automatic contract number generation for a contract.
0 = OFF (default)
1 = ON
CONTRACT_NUM_SEQ The current sequence number for contract numbering.

Resultant number is zero padded (e.g. 000001)
INFLATOR_ACTIVE Provides the capability to disable the Inflator section of contract details should they not be required.
0 = DISABLED
1 = ENABLED (default)
INVFREQ_ACTIVE Disable the invoice frequency field in the core contract details.
0 = DISABLED
1 = ENABLED (default)
TERMTYPE_ACTIVE Disable the term type field in core contract details.
0 = DISABLED
1 = ENABLED (default)
VENDOR_CAT_TITLE Permits the renaming of the Supplier Category field should the use need to be altered from its default.
VALUE_COMMENTS Comment appearing after Contract Value & Annual Contract Value fields
Default: "(ex VAT)"
VENDOR_STATUS_ENFORCED Dictates whether a contract is permitted to be added on the contract screen
according to the Supplier Status selected.
0 = DISABLED
1 = ENABLED (default)
VENDOR_LASTFINSTATUS_ENABLED Dictates whether the Financial Status field is enabled on the supplier screen.
0 = DISABLED
1 = ENABLED (default)
VENDOR_LASTFINCHECK_ENABLED Dictates whether the Last Financial Check date field on the supplier screen is enabled.
0 = DISABLED
1 = ENABLED (default)
VENDOR_FYE_ENABLED Dictates whether the Financial Year End field is enabled on the supplier screen.
0 = DISABLED
1 = ENABLED (default)
VENDOR_NUMEMPLOYEES_ENABLED Dictates whether the Num of Employees field on the supplier screen is enabled.
0 = DISABLED
1 = ENABLED (default)
VENDOR_TURNOVER_ENABLED Dictates whether the Turnover field on the supplier screen is enabled.
0 = DISABLED
1 = ENABLED (default)
VENDOR_INTCONTACT_ENABLED Dictates whether the Internal Contact field on the supplier screen is enabled.
0 = DISABLED
1 = ENABLED (default)
