Audit Log
Overview

It is important for any application to provide a history of activity so that adequate auditability is available should checks or investigations into activity need to take place. Framework provides a full Audit Log of activity to hopefully achieve this.

What is logged by Framework?

  • Users Logging On
  • Users Logging Off
  • Logon Failures
  • Account Freeze
  • Adding of any new element to Framework
  • Removal of any element within Framework
  • Update of a record.1
    • 1 In the case of an update within Framework, an Audit Log entry is made for each and every field update made. If a contract record is edited and 3 fields updated, then three update records will appear in the Audit Log providing (where possible) the pre and post edit values.
    Clear Down the Log

    Clicking the 'Cleardown Audit Log' option in the menu panel will export all the audit entries, up to the current date, to an Excel spreadsheet file and remove them from the database.

    The exported file, though remaining on the server as a backup, is emailed to whoever has been tagged in Framework's FWSetup as the Auditor.

    Audit Log User Filter

    If a particular user of Framework needs to be focused on, they can be selected from the User Filter at the top of the screen.

    On selection of the user, the audit log grid will re-populate with only the selected user's activity.

    Note: By default, ALL user activity is displayed.
