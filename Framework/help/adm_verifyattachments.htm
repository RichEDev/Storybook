Verify Attachments
Overview
The Verify Attachment screen allows an administrator to establish whether all registered attachments exist in the location specified.

Should the system be moved to a new location or server it is entirely feasible that documents may be lost or failed to be moved.

The verify screen validates all links, regardless of which area of data they are attached to. If the link is valid, a green tick () otherwise a red cross () is displayed.
Options available for invalid attachments
1. Delete Link

If the file has been left behind or cannot be located to repair the link, the link can be permanently removed. Once deleted, the attachment will no longer be referenced anywhere within the application.

2. Upload File

If the path to the file is incorrect, the file can simply be relocated using the "Upload" hyperlink. The file can then be re-uploaded to the storage location and the path entry is updated.
