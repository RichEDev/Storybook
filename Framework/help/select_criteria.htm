Defining Report Criteria
Overview

By default, any report created will report on every contract in the database.

The purpose of a report is to provide meaningful information from which either decisions can be made or important information can be communicated to others.

To make a report meaningful, it must focus on only the information that is relevant to the information requirement.

To provide this focus, Report Criteria must be specified. The more criteria specified the narrower the focus on the data becomes.

 How do I add a criteria element?

On entry to the criteria screen, any existing criteria defined against the report will be displayed. Below the existing criteria are input boxes organised in five columns.

The input boxes are as follows: -

AND / OR

Specifies the logical relationship to the other criteria. This is not required for the first line of criteria entered.

Example: Vendor = Software Europe AND Contract Product = Framework
or Contract Product = Framework OR Contract Product = Expenses

With AND, all conditions must be satisfied in order to be included in the report, whereas OR will allow either to be true for it to be included in the report.
Field Description This is the field for which the report is being asked to satisfy.
Comparison Type

This is context sensitive and will make the statement read as simply as possible. All entries will equate to 'is true' or 'is false'.

For example, date fields will provide comparison dates of 'before', 'on or after', whereas numeric fields will ask 'greater than' or 'equal to'.
Comparison Value

This is the actual value to be compared.

Depending on the type of field, this will be either a drop-down list of options or a text box where you can enter the value.
User Specified? This check box indicates that the Comparison Value is to entered at the report run time. Should this box be checked, any entry specified in the Comparison Value field is ignored.

This allows the criteria elements to by dynamic rather than static; ideal for reports that are to be published Global.

All criteria lines should be thought of as statements. For a record to be included in the report (i.e. to satisfy the criteria) it's statement must read as TRUE.

Tip: Never use AND when including two entries for the same field using equals (=) because no entity will ever satisfy both (e.g. Each contract only has a single Type selected, so it can never be Type A AND Type B).
 Common Criteria

Certain criteria options should be considered for inclusion regularly in your reports.

Archived

All Contract Status definitions equate to either a 'live' or 'archived' contract. On selection of the contract status, the 'Archived' field will be set as Y or N.

This enables easy exlusion of all 'Archived' contracts resulting in reporting of 'live' contract data only.

 Fixed (Static) Criteria

If criteria rows have the Comparison Value completed on this screen, the criteria is fixed every time the report is run.

Even if the report is published as Global, the report will execute with the same criteria applied each time. As only the Contract Owner can modify a Global report, this results in loss of flexibility and usefulness of the report.

For some fields this will be fine. For the best results and the least overhead in generating reports, these fixed criteria should be mixed with Variable criteria.

 Variable Criteria

Variable criteria means that the executor of the report will be prompted to complete the criteria item prior to the report being executed against the database.

For example, if a 'month end' report was generated, the month in question could have the date range applied as Variable criteria, thus removing the need to edit the report template each month. On running the report, the user will be prompted to enter start and end period date for which the monthly report is to retrieve data for.

It is strongly recommended that Variable criteria be implemented for reports that are to be published as Global. It is not necessary for all criteria elements to be Variable, as it may be important for some to remain static. The use of Variable criteria will add flexibility to the report you are creating.
