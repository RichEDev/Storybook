Contract Summary - Contract Detail
Overview

Contract Details is the core details of the contract or schedule of the contract.

This contains the top level information and milestone dates for the contract or schedule.

Most of the fields are self-expanatory. For full details on the standard fields available, please refer to the user guide.

Note: If the icon is displayed adjacent to a field, then a bespoke help document has been added by your administrator that corresponds to that field. To open the document, simply click the icon and the document will be opened.

Mandatory Fields

Most fields within Framework are optional. The more detail entered against a contract, the more meaningful the data is. The following exceptions to the rule are mandatory fields. These are detailed below.

Field Name Definition
Contract Description

The contract description forms part of the key definition of the contract.

The field is text and should indicate the contract's purpose or reason.
Vendor

All contracts must be associated with a Vendor. Depending on how Framework has been configured for your organisation depends on whether this field is pre-selected or not.

If adding a contract from the main menu, then a Vendor must be selected before the contract will be permitted to be saved. If adding a contract from the Vendor screen, this field will have been pre-selected.

In the case of adding contract from the menu and once contract products have been added, this cannot then be edited.

Other field information

Amongst the main contract detail, some fields are either out of the control of the user or have some additional functionality that is fundamental to the contract management.

Field Name Definition
Contract Key

The contract key is generated by Framework and cannot be modified by the user.

The prefix is determined by the Framework administrator with the remainder made up of unique database generated values. This key cannot be modified by the user.
Forecast Type

The forecast type selection (with accompanying maintenance type fields) are used to accurately calculate the forthcoming maintenance costs according to the contract or schedule.

Specific help is provided for this particular feature. Simply click the icon adjacent to the field.
Contract Dates

Contract dates are critical if the automated email review notification is to be used.

The Start and End dates of the contract indicate the period for which the contract covers.

Some contracts require a period of notification in order to cancel a contract, otherwise it will renew for a further period.
The specification of this cancellation period will produce and automatically generated cancellation date by which this would need to be actioned. The cancellation can, if you so wish, be specified manually.

The Review period permits a time period (in months) for information collection and consideration of what the next action on the contract should be.
Providing a Review period will automatically calculate the date at which the review of the contract should take place.

NOTE: This Review Date is the date at which the automatic email notification of review will commence.
Notify

The notify box displays the names of those staff who are tagged to receive the automatic review notification emails.

Clicking the 'Notify' link next to the box will move the user to the notification screen where one or more staff members can be nominated to receive the notification email.

NOTE: Notify personnel can only be added once the core contract detail has been saved to the database. Prior to that, the link will be disabled.
Contract Status

Although multiple contract states may exist, in terms of definition, the status will be either a 'Live' state or an 'Archived' state.

Live contracts are those that are currently active and will be returned as part of searches in the Summary Portal etc. Archived contract are those that are no longer active (e.g. cancelled, run their term and not renewed).

Archived contracts can remain on the database enabling them to be included in reports if and when required.

NOTE: When a contract has a status set that equates to being archived, an 'Archived Date' is stored and displayed at the side of the contract status field.
