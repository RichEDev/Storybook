User Roles
Overview
It is essential that users of the system are granted access to the areas of the system that are relevant to job they undertake in relation to Framework.

The defining of User Roles gives the ability to design profiles that categorise the different users. These roles are then assigned access rights and the areas of the application they are entitled to utilise.
Adding a role
To add a new role to the system, select the "New" on the Page Options panel.

Located: Administration > Employee Management > User Roles.

Similarly, editing an existing role is achieved by clicking the "Edit" button the role's row. The role detail screen will be displayed and the various access permissions can be chosen.
Delete a role
Clicking the delete button against a corresponding role will prompt the user to confirm the delete action. On clicking "update", the role will be permanently deleted.

Any users that are assigned the deleted role will then have their role re-assigned to the "default role".
Default role
Only one role can be designated as the Default Role.

If a Default Role exists when this option is selected on a new role, then the new role will replace the existing one, effectively removing the setting from the existing Default Role. All new users added to the system will be granted this role unless the user adding the user explicitly selects a different role.
