Vendor Status
Overview

The Vendor Status field definitions are utilised in the vendor details screen.

The key use of the field is the enforcement of proper business process. It would be bad practice to undertake any contract dealings with a Vendor whose status is not flagged internally as 'approved'. A common method of specifying a status such as this is with a RAG definition. (i.e. Red, Amber, Green).

Required Information

The primary field is the textual definition by which the Vendor Status is to be known. This is a mandatory field and duplicate definitions will be rejected.

Also provided are a sequence value and a checkbox to signify if a contract can be added.

The sequence value allows the user to dictate the order in which the definitions appear in the drop down list. It is better that a RAG status appears in the order Red, Amber, Green as this is the order of validity. Alternatively they would display in either alphabetical order or even the order in which they were added in the database (chronologically).

The checkbox signifies whether or not a new contract can be added from the vendor details screen. It therefore allows the enforcement of when and when not a new contract be added depending on the active vendor status.
