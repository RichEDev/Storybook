Administration
Overview

Administration is the area of Framework that allows the core definitions to be defined. These are divided into logical application menu sections.

A common screen is displayed, regardless of the actual elements you are viewing.

The ability to Edit or Delete an element will depend of the permission granted to the user by their role.

On entering the screen, a list of existing definitions will be displayed.

  • To Add: Click the 'New' link in the page options panel to the left of the definition list. You will be transferred to the definition screen where the details of the element can be entered.

  • To Edit: Click the 'Edit' icon () to the left of the definition. You will be transferred to the definition screen where the detail of the element can be modified.

  • To Delete: Click the 'Delete' icon () to the left of the definition. You will be prompted to confirm deletion. Clicking 'OK' on the dialog box will remove the item from the database permanently.

    • Note: The Deletion of an element will be denied if the element is assigned anywhere within the database.
     Basic Definitions

    In most cases, the elements defined are within administration are isolated definitions that have varying amounts of accompanying information. The most basic being single field text definitions.

    As soon as a definition has been added and appears in the list, it will be available for selection by a user.

     Tiered Definitions

    Tiered Definitions are those that consist of Primary and Secondary definitions.

    The Primary definition will typically be a collective description that covers the Secondary definitions that appear on selection of the Primary.

    An example is the Note Category and Type. For every Note Category, multiple Note Types can be defined. (e.g. Category of 'Recent Communication' could have secondary definitions of 'Email','Letter' or 'Telephone Call').
