Currency
Overview

Mult-currency support is an integral part of modern business. Contracts are frequently held is currencies other than the native currency of your organisation.

This area of code maintenance allows the configuration of all currencies to be used throughout Framework.

Required Information

The following elements of information are required for the Currency definition: -

Field Type Description
Currency Text This is the name of the currency by which it will be referred to throughout the application.
Currency Symbol Text The keyboard symbol / letters used to represent the currency when formatted (e.g. £,$)
Decimal Separator Text The character or symbol to be used to separate whole numbers from the decimal portion. This is '.' by default.
Decimal Places Numeric This is the number of digits (accuracy) to use for the decimal portion of the number (e.g. 2 = .00)
Group Symbol Text Character or symbol by which numbers greater than 999 have the thousands separated. This is typically a ','. (e.g. 3999 = 3,999)
Group Digits Numeric This is the number of digits (accuracy) that dictates the placement of the Group Symbol. (e.g. 3 = x,000 and 2 = x,00)
Positive Format Drop Down List Provides options on how positive currency amounts are formatted and displayed. The X represents the currency symbol.
Negative Format Drop Down List Provides options on how negative currency amounts are formatted and displayed. The X represents the currency symbol.
Rate to Base Decimal This is the amount ratio for this currency that equal 1 (one unit) of the specified base currency.

IMPORTANT: If this is the base currency, then this must be entered as '1'
Base Currency? Check Box This should only be checked if the currency specified is to be your base currency.

Note: Only ONE currency can be specified as your base currency. Selecting this check box will supercede any previous selection.

On successful update, you will be returned to the definitions list screen.
