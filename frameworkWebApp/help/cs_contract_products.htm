Contract Product Details
Contract related Products / Services
A majority of contracts have a product or service that forms the 'deliverable' of the contract.

This section permits the association of the product or service with the currently active contract.

NOTE: Only products available from the contractually selected Vendor can be added. If the required product does not appear to be available, then the Product/Vendor association should be checked in the Product definition screen.
Standard fields available
Field Name Explanation
Contract Product The drop-down list allows selection of the contract product or service. Only items available are those that are associated with the contract vendor.

NOTE: Selection of the field is mandatory.
Product Value The product value is the monetary amount on which the maintenance has been based. This may be the 'list price' of the product / service or simply the price paid.

This value is used in the 'next period maintenance' calculation if the forecast type 'Product £ v Inflator' is used.
Maintenance Value The maintenance value is the monetary amount being paid for the current contract period.

This is the value that the contractually permitted inflation amount is applied to.

NOTE: If left blank, this field will be calculated from Product Value and Maintenance Percent (assuming these values also exist).
Maintenance Percent Contracts commonly have an annual maintenance amount that is calculated as a percentage of the Product Value.

This field allows the specification of the percentage of the Product Value that forms the maintenance.

This is required when calculating contracts using the forecast type 'Product £ v Inflator'.
Projected Saving If it is possible to quantify how much a particular product or service saves your organisation, then this should be specified here.

If the Projected Saving amount is exceeded by the proposed, Next Period Maintenance, then this is highlighted in the automated email renewal notification. This acts a 'cost v benefit' indicator for the given product / service.
Currency Selecting a currency from the list permits correct management of contracts' monetary values. Without specifying a currency type, contracts will not be compared like-for-like with contracts in other currencies.
Units A contract product may have some limitation quantifier. This may be a number of users, a particular site or, in the case of mainframe software, the number MIPs.

This list allows the selection of the limitation unit if one is required.
Quantity This field simply allows the specification of the quantity of the selected Unit type covered buy the contract.
Unit Cost

Unit Cost is a monetary field enabling the ability to measure the 'cost per unit' of the contract product / service.

If left blank, this field will calculate automatically using the unit quantity selected and the maintenance value.
Other Functionality
Once contract product information has been associated against a contract, further functionality options are enabled.

Function Description
Version Registry

The functionality provided in the Version Registry is very much suited to software contracts but can be utilised for other occasions.

The Version Registry allows the tracking of multiple types of the same product or service. A good example is the WindowsT software where multiple versions may exist within an organisation but all covered by a single contract.

WindowsT may be the product, but 98/NT/2000 & XP may exist within the Version Registry. Refer to the user guide for full details.

Select the required product and click 'Version Registry' to access this functionality.
TIP: A single selector button (radio button) exists on each contract product. This indicates the current product that is to correspond to the Notes or Version Registry selection.
