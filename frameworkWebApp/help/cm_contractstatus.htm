Contract Status
Overview

It is common for contracts to be at differing stages of their life cycle. A majority are likely to be currently active and in effect, whereas others may be discontinued, expired or even in the process of being initiated.

Contract Status allows contracts to be tagged in this way. Each definition allows the specification of whether the definition implies that a contract is considered active or inactive. This then allows those contracts considered inactive to be ignored, by default, by Summary Portal queries.

Required Information

The following items of information are required for a Contract Status definition: -

Field Type Description
Contract Status Text This is the definition by which the Contract Status is to be referred.
Is Archived State? Check Box This is, by default, not checked. By checking the box you are indicating that any contract tagged with this status is considered archived / inactive / obsolete and will not be used in default searches.

On successful update, you will be returned to the definitions list screen.
