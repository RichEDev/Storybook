Reports - Column Selection
Overview

All reports must have a collection of fields defined that match the information the user aims to deliver.

The Select Columns screen allow the required fields for the report to be included or excluded depending on whether a new report is being compiled or if an existing report is being modified.

If an existing report is being modified, all current selections will be displayed. A new report will show a blank selection.

Note: All reports must be given a unique name in order to saved. The report description is optional, though will often provide a means of greater information to describe the report.

 Report Base

The report base allows the user to change the base (core) level on which the report is being generate.

The default is Contract Details, though a report can be based around Vendor Details, Product Details or Staff Details. On changing the selection in the drop down list, the fields that are available for selection change accordingly.

An example might be...

If a Contract Details based report is generated, then for each contract a single Vendor will be returned because of the relationship being one to one, whereas in a Vendor Details based report, for each Vendor reported on, multiple contract items could be returned.

All fields are grouped together under a title detailing the logical area of the application from which they originate. Clicking on the '+' icon will expand the group to show the fields available for selection.

 Select / Deselect fields

Select Single Field To select a single field, click on the field name. This should highlight the field.

Click the Select--> button. The highlighted field will move to the selected columns list.
Select Multiple Fields To the left of all available fields, a tick appears. All required fields, regardless of which grouping they are in, can be checked for selection

Click the Select Checked--> button. All checked items will be moved to the selected columns list.
Deselect Single Field To the left of each field in the selected columns list, a single select (radio) button appears. Click the radio button adjacent to the field to deselect.

Click the <--Deselect button. The field is removed from the selected list.
Deselect Multiple Fields Clicking the <--Deselect All button will remove all current selections in the list.

 Field Sequencing

Selected fields in the report will when selected singly, appear in the order required. If fields are added using the 'Select Checked' button, they will be added to the list in the order they are encountered in the list.

This is likely not to be the order required in the report.

To resequence the order of the fields: -

  • Type in the position number that you wish the column to appear

  • Click the 'Re-sequence Fields' button at the top of selected fields list.

  • Fields should be in sequence specified

    Tip: It is not necessary to provide a sequence number for every field. You may only want to specify the fields you want in position 1 and 5. All other fields will fall into place around them.
    •  Sort & Subtotal

    Each field in the selected list can be sort in ascending (A-Z,0-9) or descending (Z-A,9-0).

    Sort seeding will also be in place in a future release that will allow specification 'Sort Column A, then Column B' as is possible in spreadsheets.

    Use the drop-down list against each field to select Ascending or Descending sort for that field.

    A tick box is also available adjacent to the field which allows the user to specify whether they wish to subtotal the field. For any non-numeric field, this will be ignored.

     Saving the Report

    On completion of your report setup, you have two options. Both of which will save the report configuration (as long as a unique report name has been specified).

    Apply Criteria This option will take the user directly to the criteria definition screen. He they can specify any filter criteria to reduce the amount of returned data.

    On an existing report, any criteria already defined will be displayed.
    Run Report This option will execute the report immediately. If the report is an existing report, the report will execute with any already saved criteria.

    If it is a new report, it will run returning all data in the database.

    Tip:It is strongly recommended, for both report clarity and system performance benefit, that some criteria is always used to focus the amount of data being returned from the database.
