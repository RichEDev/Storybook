Contract Details - Inflator Metric
Overview
The purpose of the Inflator metric is to specify or cap the amount by which the cost of a contract can increase by year on year.

The most common type of Inflator Metric is the RPI (Retail Price Index).

This, and any other metric you wish to use, can be defined in the Code Maintenance section by any user with sufficient access rights.

If you do not have access to the Code Maintenance, speak to your Framework administrator and they will be able to assist.
 Forecast Type
The forecast types available are: -

Forecast Type Explanation
Product £ v Inflator Next period maintenance will be forecasted using the lesser of the Product Cost and the Current Maintenance with the selected inflator applied.
Inflator Only Next period maintenance will be forecasted using only the Current Maintenance with the selected inflator applied.

This is particularly useful where a contract is a year-on-year payment with an inflation cap applied (e.g. annual rental).
Staged Payments

Option not presently available

Allows fixed payments to be specified for certain periods of time.

This will allow payments to be forecasted of £1000, £2000, £3000 over a 3 year period.

After this period, the forecasting will resort back to the default of Product £ v Inflator.
 Maintenance Type
Typical clauses within a contract may state that: -

'the maintenance will not increase by more than the RPI or 5%, whichever is the greater.'

To accomodate clauses such as this, Framework has integrated the ability of specifying comparison calculations that allow the specification of two values and then using the correct one in the final analysis.

Standard inflation values can also be specified that then do not perform the comparison.

The maintenance types available are: -

Maintenance Type Explanation
Single Inflator (x entry only) Comparison between two values is ignored. Uses x value regardless.
Greater inflator of x & y Performs comparison between values specified in x & y boxes
and uses the higher of the two results.
Lesser inflator of x & y Performs comparison between values specified in x & y boxes
and uses the lower of the two results.
x and y Values
A drop-down list against the x and y values provides all metrics defined.

In addition to this a '%' option is available that allows the specification of just a percentage value.
This should be specified in the edit box to the right of the drop-down list.

Any inflator detail specified in y will be ignored if 'Single Inflator' is selected as the maintenance type.

TIP: The edit box to the right of the inflator is only required if the metric selected specifies an additional percentage is to be specified (e.g. RPI+%).
