User Role Detail
Overview
Each role consists of multiple permissions that can be granted or not granted to the user assigned the role.

Once the elements have been updated or selected as required, click the update button to apply the selections. To abandon modification at any time, click the Cancel button at the bottom of the screen.
Role detail and permissions
A role name must is a mandatory field and must be unique. Two role with the same name will not be permitted. The description field allow a more expansive explanation of the role to be provided.

The permissions section of the role detail are the individual permissions that can be granted to a user. Any box clicked (containing a tick) has that permission granted.

This section consists of three columns. The first is the name to the permission option followed by the check box for granting that permission. The final column contains a description briefly explaining what the permissions' actions will be.
Contract tabs accessible
The contract summary pages are divided into eight (six if recharge functionality is disabled) screen areas. Each role can be granted permission to access these sections.

Any section that the role is not granted access to will not see the section available in the navigation panel on the contract summary screen.
