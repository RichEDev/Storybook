User Setup
Overview

The User Setup screen displays an overview of all user currently defined for access to Framework.

A user must be defined before they can gain access to Framework. The level of access permissions they have is determined by the role they are assigned.

Information displayed

Element Description
Edit / Delete Clicking the edit or delete button to the left of the definition will allow modification, or removal, of the user logon.
Unfreeze If a user logon fails due to incorrect password entry consecutively in excess of the nominated allowance, the user logon is frozen for security. Clicking this 'unfreeze' button will reactivate the logon.
Login Id This is the actual logon id of the user.
Full Name A logon id may not clearly identify who the user is. The full name field allows the proper name of the user to be logged against the identity. This is required for clarification and also in the Audit Log.
Position The job title or role of the user listed.
Role Assigned The role currently assigned the to user. The role is what defines the permissions afforded to the user.
Logon Count Each time a user logs on to Framework, the counter is incremented. This is a simple method of tracking who uses Framework most frequently.

Reset counters
Logon counts provide an indication as to the level of activity of each of the users within Framework.

The counter continues unchanged until a user resets the counts. This can be done by simply clicking the Reset Counters link in the page options panel.
To Add a New User: To add a new user, click the 'New' button. You will be transferred to the User Setup Detail where full details of the new user will have to be specified.
