The Framework Logon Page
Overview

All users of Framework must access the application via the logon screen.

A valid username and password must be entered in order for the user to be successfully authenticated.

On successful authentication of a user, their allocated permissions, role and accessible locations will be obtained from the database.

If necessary, Framework can be configured to force expiry of passwords at predefined intervals. This is standard practice in many organisations and prevents users from keeping the same passwords for prolonged periods of time.

For any assistance with logging on, contact your Framework Administrator in the first instance.
Changing your password

Should a user require to change their password, they should enter their username and select the checkbox to indicate that they wish to change their password.

Additional fields will then appear below the logon panel.

  • Enter your existing password
  • Enter your new password
  • Re-enter your new password to confirm correct entry
  • Type a password hint that can be used at a later date to remind you, should it be forgotten. Note: This should not be the password itself!

    • Note: Beware that if required, the system can be configured to prevent a user from using a password in their last x number of passwords. This prevents users from simply alternating between the same two or three passwords which reduced the security of the individual.
    Forgotten your password?

    When passwords are defined, a password hint is entered that enables a reminder to be emailed to the user to help them remember their password.

    To obtain your password reminder: -

  • Type in your username
  • Click the 'Forgotten your password?' link below the password entry box.

    • Your password hint will be emailed to you.

    Still need help? If the hint still doesn't assist, your password will have to be overridden. You should contact your Framework Administrator who can do this for you.
