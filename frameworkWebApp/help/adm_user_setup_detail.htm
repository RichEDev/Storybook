User Definition Detail
Overview

This screen provides the means for an administrator to add or modify the detail for a Framework user security account.

In addition to the user detail, access can be granted or denied to any locations defined.

Note: This screen will also allow a user's password to be reset.
 Field Descriptions

The following fields are displayed for population: -

Field Name Description Mandatory /
Optional		 Max Length
Full Name This should be the 'real' name of the user. Mandatory 30
User Name This is the identity the use at the logon screen to identify themselves to the application. A user name can be any format of choice (e.g. a number, firstname or a mixture of the two). Mandatory 15
Position The job title or role within the organisation Optional 100
Password The password must be 6 to 10 characters and is case sensitive. Mandatory 10
Confirm Password Re-entry of the password enables confirmation and elimination of typing errors. Mandatory 10
Email Without an email address, the user will be unable to request a password hint should they forget it. Optional 100
Password Hint A short clue as to what their password is. This should never be the password itself!

Note: This hint will be sent via email, so will require an email address to be usable.		 Optional 50
User Role The user role selected from the list determines the actual permissions the user inherits. Individual permissions are controlled within the role definition.

Warning: Changing a role definition permission will affect all users assigned that role.		 Mandatory n/a
Icon Size Drop down list allowing user to decide whether they use the large icon display or the small icon display. Mandatory Default: Large
Location List It is possible that Framework will have multiple repository locations defined.

Each user can be granted or denied access to these locations by ticking the box adjacent to each location definition.

In addition to this, one location should be selected as the user's default location that they will access automatically on logon. This can be selected using the 'radio' (single select) button.

Note: The default location must be a location ticked for access.		 One Selection Mandatory n/a

 Reset a User's password

A common task that will arise is that of users forgetting their password.

As all passwords are encrypted within the database, it is impossible for anyone to tell a user what their password is.

If a password requires overriding: -

  • Click 'Edit' to access the user's detail.

  • Enter the new password in the 'Password' text box.

  • Re-enter the password in the 'Confirm Password' text box.

  • Click the 'Update' button.

    • The user should be advised to change their password as soon as possible. The override password should be considered as temporary.

    Note: On reporting a forgotten password, the user should be reminded of their password hint as a first try. This can be received via email by clicking the 'Forgotten Password?' link on the logon screen.
