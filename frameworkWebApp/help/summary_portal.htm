Overview
By default every time you log in to FRAMEWORK, you will be presented with the Summary Overview Screen.

At the top of the screen is a status bar, which indicates which screen you are in at all times. Displayed in the centre are your personal details followed by today's date.

Once the Framework application is open, the core configuration data elements should have been agreed and now are ready to be entered into the Table Maintenance and Code Maintenance before the entry of live information can proceed. The core elements of information are the parts of the software that configure it to the personal requirements of the individual customer installation.

Dependant on the Access Permissions granted, enter the Administration,Table Maintenance & Code Maintenance screen select from the Tool Bar on the left hand side. There is a help button on each page to explain each feature.
Quick Access

Quick Access Explanation
Supplier Name Use this box to locate all Suppliers containing the specified letter(s) that are in the name. This is treated as a Wildcard and, as such, can appear anywhere in the supplier name.
Last Report Used Remembers the last report accessed from individuals last log in.
Last Contract Accessed Remembers the last contract accessed from individuals last log in.
Contract Quick Search
Explanation
Contract Key This is a unique number generated by the database. The prefix characters are customisable to suit an organisation.
Contract No

Specific contract reference number that exists on the contract.

This may be dictated by one party or the other and is likely to differ from the Contract Key generated by Framework.

Description Use this box to locate all contracts containing the specified letter(s) that are in the name.

This is treated as a Wildcard and, as such, can appear anywhere in the contract name.
Product

Use this box to locate all products containing the specified letter(s) that are in the name.

This is treated as a Wildcard and, as such, can appear anywhere in the product name.
Contracts Category Locate list of contracts allocated to a particular category.
Invoice Number

Returns all occurrences of a particular invoice number held within the system.

Link to the related contract is returned and clicking the link shall transfer the user directly to the contract and place them directly on the invoice section where the required invoice is held.
Purchase Order (PO) Number

A Purchase Order (commonly abbreviated to PO) can be allocated against either an invoice or an invoice forecast.

Both invoice and forecasts will be searched for the requested PO Number.
